# 交付目录规则

这套规则只管对外交付，不影响系统运行层。

## 基本原则

1. 每个链接、每个批次、每个独立委托，单独生成一个交付文件夹。
2. 交付文件夹统一放在 `D:\CodeX\paper-download-grade\case_exports`。
3. 交付文件夹由系统运行层复制生成，不直接移动原始文件。
4. 如果同名交付文件夹需要重做，旧目录先备份，再生成新目录。

## 老师与链接映射规则

1. 一个固定链接对应一个老师的一个阶段任务。
2. 后续同一个链接再次出现时，默认继续归入同一个老师的同一个阶段。
3. 当前阶段默认使用 `初稿`，如果以后切到终稿，再单独登记成新的阶段。
4. 交付目录名称统一使用 `老师名_阶段`，例如 `牛逼老师_初稿`、`der老师_初稿`。

## 目录结构

每个交付文件夹固定包含以下 4 个目录：

- `01_论文初稿`
- `02_评分结果`
- `03_学生评语`
- `04_汇总`

## 各目录用途

`01_论文初稿`

- 保存下载下来的论文初稿副本。
- 按学生单独建子目录，避免重名和混淆。

`02_评分结果`

- 保存该学生的评分报告、结构化结果、视觉审稿 PDF 等证据链。
- 这是老师和系统复核用的，不直接给学生看。

`03_学生评语`

- 保存可以直接发给学生的修改意见。
- 统一使用老师口吻的自然语言，不使用内部评分术语。

`04_汇总`

- 只放最终用户真正需要看的总表。
- 汇总表只保留这些核心信息：学号、姓名、论文初稿位置、分数、教师评语。
- 不在总表里放中间日志、内部诊断路径、调试信息。

## 命名规则

1. 交付文件夹名称应当对应来源或委托，例如：`张老师_第一批`、`20260317_链接A`。
2. 学生目录统一优先使用 `学号_姓名`。
3. 同一学生的文件保持同名映射，方便从总表反查原稿、评分结果和评语。

## 安全规则

1. 先复制，后核对，最后再决定是否清理旧交付包。
2. 不在运行层直接移动或改名核心原始文件。
3. 任何覆盖操作都必须先生成备份目录。
